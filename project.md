# Ретро-Аркада (arcade-hub)

## Что это

Веб-платформа с оригинальными браузерными играми в духе аркадных автоматов
XX века и таблицами лидеров. Никакой регистрации: игрок вводит ник (3–12
символов, латиница/цифры) только в момент отправки рекорда — как ввод имени
на аркадном автомате после game over.

## Для кого

Для тех, кто хочет за 30 секунд начать играть в простую, ностальгическую
игру в браузере — без аккаунта, без установки, без слежки за личными
данными — и посоревноваться за место в таблице лидеров.

## Функции

- Набор оригинальных мини-игр классических жанров (змейка, сапёр, посадка
  модуля, «толкай ящики», пасьянс) — свои названия, своя графика, свои звуки.
- Ввод счёта в таблицу лидеров по нику, без пароля и аккаунта.
- Серверная проверка правдоподобности счёта (токен сессии + ограничение
  максимально возможного прироста очков в секунду) — клиенту в вопросе
  честности счёта не доверяем.
- Топ-100 в таблице лидеров на игру, простая защита от XSS/некорректных
  ников.
- Rate limiting на отправку счёта и старт сессии по IP.

## Чего не будет

- Регистрации, паролей, e-mail, хранения любых персональных данных.
- Чужих ассетов, спрайтов, звуков — весь контент оригинальный.
- Игр, копирующих чужие торговые марки (например, клона «Тетриса» в
  классическом виде с полем 10×20 — это trade dress).
- Доверия к счёту, присланному клиентом «как есть», без серверной проверки.
- Глобального движка на старте: общий код (`core/`) выносится только когда
  появляется вторая игра, не раньше.
- Соцфич (комментарии, чат, друзья) и нативных мобильных приложений — только
  браузер.

## Результаты исследования (5 направлений)

*Реконструкция без точного текста промпта из урока — при наличии оригинала можно пересчитать точнее.*

**Конкуренты.** Прямых конкурентов немного именно в нише «без регистрации,
только ник + рекорд»: большинство ретро-аркад-сайтов либо перегружены
рекламой и чужими ROM/эмуляторами (юридически рискованно), либо требуют
аккаунт для лидерборда. Ниша — честная, лёгкая, юридически чистая аркада
с открытым порогом входа — свободна.

**Функции.** Ключевая функция уже есть (игра + лидерборд без аккаунта).
Следующие по ценности, не первой очереди: несколько игр на одной платформе
(общий `core/` появится с игрой №2), общий "хаб" с превью игр, история
последних результатов игрока в localStorage (не на сервере — без личных
данных).

**UX.** Главный риск UX — доверие к честности лидерборда: если игроки
увидят абсурдные счета вверху таблицы, доверие к платформе упадёт мгновенно.
Поэтому анти-чит (серверная валидация счёта) — это не техническая деталь,
а требование к продукту. Второй момент — мгновенный старт без трения
(без форм регистрации), это и есть основное УТП.

**Технологии.** Минимальный стек (Express + `node:sqlite` + vanilla JS)
оправдан: сайт лёгкий, не требует SPA-фреймворка, а простота ускоряет
итерации на этапе одной игры. Пересмотреть стек имеет смысл только если
игр станет много и понадобится общий рендер-движок — то есть снова вопрос
для стадии `core/`, не для старта.

**Монетизация.** Явной монетизации на старте нет и не планируется как
приоритет: платформа без аккаунтов плохо совместима с подписками/платным
контентом. Реалистичный вариант в будущем — ненавязчивая статичная реклама
между играми, без трекинга пользователей (иначе конфликт с принципом
«без персональных данных»). Не блокирует MVP.

## Заметки после критического прохода

- **Найдено и исправлено:** в изначальном ТЗ был пункт «базовый фильтр
  недопустимых ников», но в коде проверялись только длина и разрешённые
  символы — самого фильтра не было. Добавлен минимальный список
  запрещённых подстрок в `scoreRoutes.js`.
- **Известное ограничение (не исправлено, осознанно отложено):** игра
  «Ужик» управляется только клавиатурой (стрелки/WASD) — сенсорного
  управления для телефонов пока нет. Для MVP с одной игрой это приемлемо,
  но если платформа позиционируется как «браузерная» в широком смысле
  (включая мобильный браузер), это стоит учесть при выборе второй игры
  или доработать отдельным шагом.
- Ники не уникальны (два игрока могут занять один и тот же ник) — это
  осознанное решение, как на аркадных автоматах с вводом инициалов, а не
  недосмотр.

## Текущий статус

Первая игра («Ужик», механика змейки) реализована end-to-end: игра →
отправка счёта → серверная валидация → таблица лидеров → запуск.
Технологии: Node.js + Express, `node:sqlite` (встроенный модуль, без
нативной сборки), ванильный JS/Canvas на клиенте, без фреймворков и сборки.
